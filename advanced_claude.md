# 서비스 개선 계획 (Advanced)

현재 RAG 챗봇은 동작 가능한 상태(Q&A 10,801건 + 규정 168 + 매뉴얼 270 = 약 11,240건)이며, 이 문서는 그 다음 단계의 개선 작업 후보를 정리한 계획서입니다.

---

## 먼저 결정할 방향

서비스를 어디까지 키울지에 따라 우선순위가 달라집니다.

1. **개인 도구**: 본인/소규모 팀 검색 도우미 → 품질·UX 위주
2. **사업단 내부 서비스**: 사업단 직원·교수·대학원생 배포 → UX·운영·인증 추가
3. **외부 공개 서비스**: 다른 사업단도 사용 → 평가·안전성·인프라 본격화

---

## A. 측정 — 가장 큰 공백, 1순위 권장

지금은 개선의 효과를 "체감"으로 판단 중. 숫자로 비교할 토대가 필요.

- **A1. 평가 데이터셋**: 질문 30~50개 + 정답·근거 페어. (노력 0.5~1일 / 효과 핵심)
- **A2. 자동 평가 스크립트**: retrieval@k, citation faithfulness, 답변 정확도. (0.5일 / 큼)
- **A3. 요청 로그 저장**: 질문·검색결과·답변·비용을 SQLite/JSONL. (2시간 / 중)
- **A4. 피드백 버튼 (👍/👎)** + 로그 연동. (2시간 / 중)

## B. 검색·답변 품질

- **B1. 하이브리드 검색** (BM25 + dense): 한국어 도메인 키워드 매칭 강화. (1~2일 / 중~큼)
- **B2. Cross-encoder re-ranker**: top-20 후보 재정렬. (1~2일 / 큼)
- **B3. Query rewriting**: LLM이 사용자 질문을 도메인 키워드로 확장. (0.5일 / 중)
- **B4. 임베딩 모델 업그레이드**: `text-embedding-3-small` → `large` (재임베딩 ~$1). (30분 / 중)
- **B5. LLM 업그레이드**: gpt-4o-mini → gpt-4o 또는 Claude. (10분, 비용 ~6배 / 중~큼)
- **B6. 인용 검증 후처리**: LLM 인용 citation이 실제 컨텍스트에 있는지 자동 확인. (0.5일 / 중)
- **B7. 카테고리 분류 + 사이드바 필터**: 장학금/국제화경비/참여자격 등. (1~2일 / 중)

## C. 사용자 경험 (UX)

- **C1. 멀티턴 대화**: 이전 답변을 다음 질문 컨텍스트에. ("그럼 외국인은요?"). (0.5일 / 큼)
- **C2. NRF 원본 링크**: 인용 nttId 클릭 시 게시판 이동. (1시간 / 중)
- **C3. 답변 정지·재생성** 버튼. (2시간 / 작)
- **C4. 자주 묻는 질문 추천**: 입력창 위에 클릭형 예시. (2시간 / 중)
- **C5. 답변 복사·공유** 버튼. (1시간 / 작)
- **C6. 다크모드·모바일 반응형** 손질. (2시간 / 작)

## D. 운영·인프라

- **D1. 정기 크롤 + 자동 임베딩** (cron / GitHub Actions 일 1회): 신규 Q&A 자동 반영. (0.5일 / 큼)
- **D2. 답변 캐싱**: 같은 질문 재호출 시 캐시 사용. (0.5일 / 중, 비용↓)
- **D3. Docker화 + 클라우드 배포** (Streamlit Cloud / Cloud Run / Fly.io). (1일 / 외부 공개시 필수)
- **D4. 간단 인증** (Google OAuth, 도메인 화이트리스트). (0.5일 / 내부 배포시 필수)
- **D5. API 키 secret manager**: `.env` → 클라우드 secret store. (30분 / 보안)
- **D6. 사용량·비용 대시보드**. (0.5일 / 중)

## E. 콘텐츠·확장

- **E1. 첨부파일 OCR/추출**: BK21 게시판 첨부 한글·PDF 임베딩. (2~3일 / 중~큼)
- **E2. NRF 공식 FAQ 페이지** 추가 크롤. (0.5일 / 중)
- **E3. 다른 정부 R&D 사업 규정** 추가 (G-BK, 글로컬 등). (데이터 양에 따름)
- **E4. 사업단별 자체규정 분리 운영**. (사업단 수만큼)

---

## 추천 진행 순서 (방향 결정 전 default)

1. **1단계 (1~2일) — 측정 토대**: A1 + A2 + A3
   - 이후 모든 개선 효과를 숫자로 비교 가능
2. **2단계 (1~2일) — 가성비 품질 향상**: B6 (인용 검증) + B3 (query rewrite) 또는 B5 (LLM 업그레이드)
3. **3단계 — 방향 결정 후**
   - 외부 공개 → D1·D3·D4
   - 내부 도구 → C1·C2 + D1
   - 본격 R&D → B1·B2 + E1

---

## 다음 액션 제안

- 방향(1·2·3)을 정하면 그에 맞춰 첫 1~2단계를 좁혀 즉시 착수.
- 결정이 어렵다면 **A1 (평가 데이터셋)부터 시작**을 권장 — 어떤 방향으로 가도 필요한 토대고, 0.5일이면 1차 버전이 나옴.
