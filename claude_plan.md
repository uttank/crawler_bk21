## 0. 현재 상태 (출발점)

| 항목 | 현황 |
|---|---|
| 데이터 | Q&A 10,801건 + 규정 168 + 매뉴얼 270 = **약 11,240** 문서 |
| Chroma 컬렉션 | 단일(`bk21_qna`) + `doc_type` 메타로 분기 |
| 검색 | 규정/Q&A 별도 쿼리 후 균형 결합, Q&A에만 답변일 페널티 |
| 임베딩 | `text-embedding-3-small` (1536-dim) |
| LLM | `gpt-4o-mini`, temperature 0.1, 스트리밍 |
| 정제 | 줄 단위 인사말 제거 (substring 방식 폐기), `Answer_Date` 추출, `Combined_Text` 임베딩 |

| UI | Streamlit, 출처 expander 두 영역 분리 (📖 규정 / 💬 Q&A) |
| 측정 | **없음** — 개선 효과를 체감으로만 판단 |
| 운영 | `.env`에 API 키, 로컬 실행, 자동 갱신·인증·배포 미구축 |

## 1. 방향 결정 (필수 선결)

서비스가 누구를 위한 것인지에 따라 우선순위가 크게 달라집니다. 아래 셋 중 답을 정해주세요.

1. **개인 도구** — 본인/소규모 팀 검색 도우미 → 품질·UX 위주
2. **사업단 내부 서비스** — 사업단 직원·교수·대학원생 배포 → UX·운영·인증 추가
3. **외부 공개 서비스** — 다른 사업단도 사용 → 평가·안전성·인프라 본격화

방향이 정해지기 전 default 추천: **Phase A → Phase B 일부**(아래 §7 참조).

---

## 2. Phase A — 측정 토대 (1순위)

지금 가장 큰 공백. 모든 후속 개선의 전제 조건.

- **A1. 평가 데이터셋(Golden Set)** — 질문 30~50개 + 정답·근거(`nttId` / 조문 citation) 페어. *(0.5~1일 / 효과 핵심)*
- **A2. 자동 평가 스크립트** — `retrieval@k`(정답 문서가 top-k에 존재하나) + citation faithfulness + 답변 정확도. RAGAS 도입은 A2의 확장 단계로. *(0.5일 / 큼)*
- **A3. 요청 로그** — 질문·검색결과·답변·토큰비용을 SQLite/JSONL에 구조적 로깅. *(2시간 / 중)*
- **A4. 사용자 피드백 버튼(👍/👎)** + A3 로그 연동. *(2시간 / 중)*

---

## 3. Phase B — 검색·답변 품질

A1 평가셋 위에서 측정하며 진행.

- **B1. 인용 검증 후처리** — LLM이 인용한 citation 문자열이 실제 컨텍스트에 존재하는지 자동 확인 → 환각 방지. **가성비 최상**. *(0.5일 / 중~큼)*
- **B2. 하이브리드 검색 (BM25 + dense)** — `rank_bm25` 또는 Chroma `where_document` 키워드 필터와 dense 결과 결합. 한국어 도메인 키워드("국제화경비", "참여대학원생")에 강함. *(1~2일 / 중~큼)*
- **B3. Cross-Encoder 리랭커** — top-20 후보를 `BGE-Reranker` 또는 `Cohere Rerank`로 재정렬. *(1~2일 / 큼)*
- **B4. Query Rewriting / 확장** — 사용자 질문을 LLM이 도메인 키워드로 재작성. **Multi-Query**(여러 변형 검색 후 합치기), **HyDE**(가상 답변 임베딩 검색)도 같은 계열. *(0.5~1일 / 중)*
- **B5. 임베딩 모델 업그레이드** — `text-embedding-3-small` → `large` (1만 건 재임베딩 ~$1, 한국어 retrieval 정확도↑). *(30분 / 중)*
- **B6. LLM 업그레이드** — gpt-4o-mini → gpt-4o 또는 Claude. (10분, 응답 비용 ~6배 / 중~큼)
- **B7. 카테고리 분류 + 사이드바 필터** — 장학금/국제화경비/참여자격 등 자동 분류 후 검색 범위 좁히기. *(1~2일 / 중)*

## 4. Phase C — 사용자 경험 (UX)

- **C1. 멀티턴 대화** — Conversation Summary Buffer로 직전 답변을 다음 질문 컨텍스트에 포함 ("그럼 외국인은요?"). *(0.5일 / 큼)*
- **C2. NRF 원본 링크** — 인용 `nttId` 클릭 시 BK21 게시판 이동. *(1시간 / 중)*
- **C3. 답변 정지·재생성 버튼.** *(2시간 / 작)*
- **C4. 자주 묻는 질문 추천** — 입력창 위에 클릭형 예시. *(2시간 / 중)*
- **C5. 답변 복사·공유 버튼.** *(1시간 / 작)*
- **C6. 다크모드·모바일 반응형 손질.** *(2시간 / 작)*

## 5. Phase D — 운영·인프라

- **D1. 정기 크롤 + 자동 임베딩** — cron / GitHub Actions 일 1회. 신규 Q&A 자동 반영. *(0.5일 / 큼)*
- **D2. 답변 캐싱** — 같은 질문 일정 시간 내 재호출 시 캐시. (※ 임베딩 캐싱은 별도. 둘 다 도입 권장) *(0.5일 / 중)*
- **D3. 임베딩 캐싱** — `text → vector` 로컬 캐시(SQLite/diskcache). 재처리 시 비용 절감. *(2시간 / 중)*
- **D4. API 재시도 정책** — `tenacity`로 OpenAI Rate Limit·네트워크 오류 자동 복구. *(2시간 / 중)*
- **D5. 인덱스 manifest** — 임베딩 모델·차원·인덱싱 일자를 `manifest.json`에 기록 → DB와 모델 버전 일치 보장. *(1시간 / 작)*
- **D6. Docker화 + 클라우드 배포** — Streamlit Cloud / Cloud Run / Fly.io. *(1일 / 외부 공개시 필수)*
- **D7. 간단 인증** — Google OAuth, 사업단 도메인 화이트리스트. *(0.5일 / 내부 배포시 필수)*
- **D8. Secret manager** — `.env` → 클라우드 secret store. *(30분 / 보안)*
- **D9. 사용량·비용 대시보드** — 간단한 Streamlit 페이지. *(0.5일 / 중)*

## 6. Phase E — 콘텐츠·확장

- **E1. 첨부파일 OCR/추출** — BK21 게시판 첨부(한글 파일·PDF) 본문 임베딩. *(2~3일 / 중~큼)*
- **E2. NRF 공식 FAQ 페이지** 추가 크롤. *(0.5일 / 중)*
- **E3. 다른 정부 R&D 사업 규정** 추가 (G-BK, 글로컬 등). *(데이터 양에 따름)*
- **E4. 사업단별 자체규정** 분리 운영. *(사업단 수만큼)*
- **E5. 매뉴얼 PDF 청킹 정밀화** — 현재 `[큰섹션]` + 길이 기반인데, 디지트 분리 보정(텍스트 후처리)으로 sub-section 인용까지 가능하게. *(1~2일 / 중)*
- **E6. 데이터 정제 회귀 모니터링** — 전처리 전후의 문자수·보일러플레이트 제거 비율을 리포트로 출력하여 정제 부작용 감지. *(2시간 / 작)*
- **E7. 근사 중복(Near-duplicate) 필터** — 질문 유사도 기반 중복 제거. 현재 nttId 기반 dedup은 **Chroma upsert가 우연히 처리하는 수준**이라 명시적 단계로 추가. *(0.5일 / 작~중)*

## 7. 추천 진행 순서

**default (방향 미결정 시)**

1. **A1 + A2 + A3** — 측정 토대. (1~2일)
2. **B1 + B4** — 인용 검증 + Query Rewrite. 가성비 최고. (1일)
3. 여기까지 마치면 평가셋으로 효과를 숫자로 확인 가능. 이후 방향 결정.

**개인 도구 노선**

- B5 (임베딩 large) → B6 (gpt-4o) → C1 (멀티턴) → C2 (원본 링크)

**사업단 내부 서비스 노선**

- A1~A4 → C1·C2·C4 → D1 (정기 크롤) → D7 (인증) → D6 (배포)

**외부 공개 노선**

- A1~A4 → B1·B2·B3 (품질) → D1·D4·D5 → D6·D7·D8 → D9 (대시보드) → 안전성 검토
